Use Of Pickle to Store and Retrive data


Storing and retrieving is a crucial task to avoid re-processing for same data again. Mostly we can found data in .csv format so you might fimiliar with pandas.

Problems with Pandas

You have two statements, each with diffrent number of words, represented as below.

In [30]:
ls
Out[30]:
[['Hi,', 'I', 'am', 'Jay.'], ['This', 'is', 'Jupyter']]
In [31]:
import pandas as pd
df = pd.DataFrame(ls)
df
Out[31]:
0 1 2 3
0 Hi, I am Jay.
1 This is Jupyter None

Can you see that "None". That's the problem and a huge problem while working with oddly distributed data. It consumes so much memory as well as add read, write overhead.


Issues with Pandas

  • Take More memory
  • Slower while Storing/retrieving data ( You can only feel thas while working with data of size 100 or more MBs )
  • Can't handle multiple datatypes ( might change datatype )
  • No functionality to handle multidimentional data like image without distorting actual format.


Pickle:

A very powerful library, It preserves the data as it's name suggests.

In [41]:
import pickle
f = open("test",'wb')
pickle.dump(ls,f)

'wb' : stands for Write Binary
Now you can see a file named "test" in the same folder of your program file.

In [42]:
o = open("test",'rb')
data = pickle.load(o,encoding='bytes')
In [43]:
data
Out[43]:
[['Hi,', 'I', 'am', 'Jay.'], ['This', 'is', 'Jupyter']]

'rb' : stands for read Binary
We had retrived our actual data in same form as before.

For the user of python 2.x.x use cPickle instead of pickle it is almost 1000 faster. Python 3.x.x already imports faster version if available.


How it works:

pickle serializes the python obejects to binary and store as it it, for Example to store csv we might need different definition of datatype for each column, but if datatype is same pickle will store only one definition.

Inshort pickle stores data in same format as in running program.


An Experiment:

  • Take a big numpy array ( Aroudly take 1000 x 1000 x 1000 entries in any form for actually feel difference. )
  • Dump it with Pickle and store it with Pandas
  • Read with both and compare the speed and simplicity



Conclusion

Pickle is useful when it comes to just storing large data, it can do it in seconds, In my experience it can store 4 Gigs of data in just 15 seconds. Good luck with experiments....
